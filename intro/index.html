    What is JellyFaaS

    Note

    Depending on your use-case, you might only engage with some aspects of JellyFaaS. Because of this, this page will provide a broad overview of all aspects of the platform, while seperate guides and tutorials will cover specific sections in more detail.

    JellyFaaS is a collection of AI-ready cloud functions, tools, and SDKs enabling developers to use community-driven code to speed up their workflow and reduce time to delivery, technical debt, and infrastucture costs.

    More specifically, JellyFaaS is composed of 3 main elements:

    Cloud Functions


    Anyone can develop and deploy their own cloud functions to JellyFaaS, for use by the entire community. JellyFaaS functions are automatically hosted, scaled, and managed by us. Functions can be written in many popular languages and consumed via REST or our SDKs.

    AI Framework


    JellyFaaS functions can be used to augment popular LLMs with live, contextual awareness and actions, such as looking up current weather data or sending an email. JellyFaaS also provides tools for using custom datasets with LLMs, to further adapt AIs to developers' use-cases.

    Tools and SDKs


    The JellyFaaS CLI enables developers to deploy functions to the platform instantly. JellyFaaS also provides SDKs, making it easy to use functions and AI capabilities in many supported languages.

    How does JellyFaaS work?

    Functions

    At it's core, JellyFaaS is a platform for developers to deploy and use functions. In this context, a "function" is essentially a small, isolated, and stateless web service that "does one thing only" by handling simple HTTP requests. For example, a function might process an image file or send an email.

    Functions automatically scale as needed and can be written in variety of languages, including:

    • NodeJS
    • Python
    • Go
    • C#
    • Java
    • Ruby
    • PHP

    Functions can be deployed via the JellyFaaS REST API, but it is recommended to use the CLI tool to streamline this process. A simple specification file defines the function's usage, both acting as documentation and enabling AI integration. Behind the scenes, deploying a funciton performs validation of your specification, containerizes the app,

    After deployment, functions can invoked via the JellyFaaS REST API or SDKs, regardless of the language the function was written in. For example, a function written in Ruby could be invoked by a client via the NodeJS SDK.

    AI Function Calling

    JellyFaaS specification files have been designed to completely define the signature of a function. This serves as documentation, but also enables them to be used by large language models such as GPT and Gemini.

    Using the JellyFaaS REST API and SDKs, you can specify functions with your LLM prompts. The LLM may decide to invoke the specified functions to help answer your prompt.
