    JellyFaaS SDK Guide

    This guide provides a comprehensive overview of the JellyFaaS SDK, including installation, environment setup, and executing your first API call.

    For the complete class reference, see Python Class Reference

    1. Installation

    The JellyFaaS SDK is available on PyPI. To install, use the command appropriate for your operating system:

    py -m pip install jellyfaas

    python3 -m pip install jellyfaas

    2. Overview

    The JellyFaaS SDK simplifies interaction with the JellyFaaS API by providing a Client class to manage authentication and streamline requests. An API key is required to use the SDK. If you do not have one, refer to the Getting an API Key section.

    API Credits

    The functions used in this guide do not consume API credits.

    See SDK Introduction for more details.

    3. Environment Setup

    It is recommended to store your API key as an environment variable to avoid hardcoding sensitive information in your scripts.

    4. Client Setup

    4.1 Creating a ClientConfig Object

    First, create a ClientConfig object that stores your API key and manages authentication:

    import os
from jellyfaas import ClientConfig

api_key = os.getenv("JELLYFAAS_API_KEY")

if api_key is None:
    raise ValueError("API key is not set in environment variables")

config = ClientConfig(api_key=api_key, debug=True)

    4.2 Initializing the Client

    Next, create a Client instance using the ClientConfig object:

    from jellyfaas import Client

client = Client(config=config)

    5. Executing a Function Call

    The following example demonstrates how to execute an API call to the helloworld function, passing query parameters and a JSON request body.

    5.1 Function Lookup

    To interact with a function, you must first look it up by its ID (short name):

    client.lookup_function("helloworld")

    This informs the Client instance of the function's request and response requirements, including method (GET, POST, etc.), query parameters, and body.

    5.2 Setting the Request

    The helloworld function requires two query parameters: firstname and lastname. You can set them as follows:

    query_params = {
    "firstname": "John",
    "lastname": "Doe"
}

    The request body for the helloworld function includes a message and a rating. You can define it as a dictionary or as a class instance:

    request_body = {
    "message": "JellyFaaS is cool",
    "rating": 5.0
}

    class Review:
  def __init__(message, rating):
    self.message = message
    self.rating = rating

request_body = Review("JellyFaaS is cool", 5.0)

    With the query parameters and request body defined, you can set the request on the client:

    client.set_request(query_params, request_body)

    5.3 Invoking the Function

    Finally, invoke the function:

    client, result = client.invoke()
print(result)

    5.4 Complete Example

    Here is the complete example for invoking the helloworld function with query parameters and a request body:

    import os
import jellyfaas

api_key = os.getenv("JELLYFAAS_API_KEY")
config = jellyfaas.ClientConfig(api_key=api_key, debug=True)

query_params = {
    "firstname": "John",
    "lastname": "Doe"
}

request_body = {
    "message": "JellyFaaS is cool",
    "rating": 5.0
}

client, result = (
    jellyfaas.Client(config)
    .lookup_function("helloworld")
    .set_request(query_params, request_body)
    .invoke()
)

print(result)

    5.5 Function Chaining

    All of the Client class's functions return the Client instance, enabling function chaining. The only exception to this is Client.invoke(), which returns a tuple of the client instance, and a response object. Using this functionality, the above example could be rewritten as: 

    import jellyfaas
import os
api_key = os.getenv("JELLYFAAS_API_KEY")

# Create client config instance
config = jellyfaas.ClientConfig(api_key)

# Create Client instance, lookup cloud function, set the request params/body, and invoke.
client, response = jellyfaas.Client(config).lookup_function("helloworld").set_request({"name": "arthur"}, {"order": ["ham", "jam", "spam"]}).invoke()

print(response)

    6. Response Handling

    The SDK allows you to automatically map the response JSON to a specified class. This is useful for managing complex responses.

    Example with Custom Response Class

    class MyResponse:
    def __init__(self):
        self.message = None
        self.status = None

client.set_response(MyResponse)
client, result = client.invoke()
print(result.message, result.status)

    7. Error Handling

    The JellyFaaS SDK provides custom exceptions for different error scenarios. Below is an example demonstrating how to handle these exceptions:

    import jellyfaas
import os

api_key = os.getenv("JELLYFAAS_API_KEY")

try:
    config = jellyfaas.ClientConfig(api_key=api_key)
    client, result = (
        jellyfaas.Client(config)
        .lookup_function("helloworld")
        .set_request(query_params=query_params, request_body_instance=request_body)
        .invoke()
    )
    print(result)
except jellyfaas.AuthenticationFailedException as e:
    print(f"Authentication failed: {e}")
except jellyfaas.FunctionLookupException as e:
    print(f"Function lookup failed: {e}")
except jellyfaas.SetRequestException as e:
    print(f"Setting request failed: {e}")
except jellyfaas.InvocationException as e:
    print(f"Function invocation failed: {e}")

    8. Debugging

    Both ClientConfig and Client classes utilize Python's built-in logging module. Ensure that your application is configured to capture these logs:

    import logging

logging.basicConfig(level=logging.DEBUG)
